# 挂载与挂载点

难度：7

## 挂载

在做完分区后，需要把分区挂载到/mnt上。通常，分区1是启动分区，而分区2是主分区。

在实际操作中，要先挂载主分区，再挂载启动分区，最后检查挂载点。

    mount /dev/vda2 /mnt
    mkdir /mnt/boot
    mount /dev/vda1 /mnt/boot
    mount | grep /dev/vda2
    mount | grep /dev/vda1

## 挂载点

为什么需要检查挂载点？这恰恰是新手最需要问的问题之一。假想一下，如果挂载点在关键时刻“掉链子”，那么后续安装的时候就需要漫无目的地寻找失败的原因；又因为它是元发行版，和别的发行版不一样，没有GUI的辅助，我们无法知道挂在哪里（除非在清华源中下载archlinux-gui）。所以检查挂载点是第一大“硬骨头”。

`arch-chroot`会自动检测虚拟文件系统的挂载情况并挂载虚拟文件系统。

在/mnt目录下执行下述命令，可手动挂载虚拟文件系统：

    #!/bin/bash
    # 创建核心虚拟文件系统挂载点（若不存在）
        mkdir -p /proc /sys /dev /run /tmp

    # 挂载虚拟文件系统（无输出）
    mount -t proc proc /proc                  # 进程信息文件系统
    mount -t sysfs sysfs /sys                 # 内核设备/驱动文件系统
    mount -t devtmpfs devtmpfs /dev           # 动态设备管理文件系统
    mount -t tmpfs run /run                    # 运行时临时文件系统
    mount -t tmpfs tmpfs /tmp -o mode=1777     # 临时文件存储（带粘滞位）
